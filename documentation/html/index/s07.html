Summary

The Moving Window library works with data that conforms to the Raster Concept. Geographical raster layers carry information of the spatial distribution of some variable. The data is stored in a matrix that corresponds to a regular grid and where the each cell, or pixel, corresponds to a square square geograpical area. To convert from matrix indices to geographical coordinates it is necessary to know the orientation of the grid, its corner coordinates and the size of each cell. The moving window library is not concerned with aspects of geographical representation, but only considers the matrix of pixel values.

Thus a Raster is not more than a matrix. The operation on Raster that are supported are to query the dimension of the matrix and to access values.

Range

Summary

Summary

The key objective of this library is to make rasters available as ranges. However, the C++ concepts of ranges and iterators are still in flux and the current ones not very useful for iteration with proxy references (http://ericniebler.com/category/ranges/). Therefore the concepts are used in a rather lose sense. Hopefully in the future there will be useful concepts for ranges and iterators that we can adapt to.

In the meanwhile, keep in mind that the iterators and ranges used here are useful for iterating over raster datasets, but do not conform to STL concepts, and should not be used with STL algorithms, like std::sort.
